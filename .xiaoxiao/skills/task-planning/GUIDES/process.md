# 流程详解 | task-planning

## Step 1: 任务拆解

**目标**: 从 UI 和架构文档拆解出具体任务

**做法**:
- EPIC = 大功能块
- Task = 可在 1-3 天完成的工作
- 每个 Task 有明确输出

---

## Step 2: 任务估算

**做法**:
- 相对估算（Story Points）
- 或绝对估算（人天）

**注意**:
- 考虑不确定性
- 留 Buffer

---

## Step 3: 依赖排序

**做法**:
- 按技术依赖排序
- 尽早识别阻塞点
- 并行任务最大化

---

## Step 4: 优先级确认

**原则**:
- P0 = 核心功能，不做不成
- P1 = 重要功能，后续可加
- P2 = Nice to have

---

## 输出格式

```markdown
# 任务列表

## EPIC 1: [功能块]
### Task 1.1: [任务名]
- 估算：X 人天
- 输出：
- 依赖：
### Task 1.2: ...

## EPIC 2: ...
```

---

## CONFIRM 话术

> "共 {N} 个任务，预计 {X} 天。
> P0: {N0} 个
> P1: {N1} 个
> P2: {N2} 个
> 确认任务排序吗？"
