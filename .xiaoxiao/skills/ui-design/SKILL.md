**前置条件**: architect 已完成

**核心动作**:
1. 信息架构与结构设计
2. 交互流程设计
3. UI稿图生成（低保真）
4. 输出设计文档

---

## Layer 2 - 上下文

### 目标

将产品需求转化为可触摸的界面设计方案。

### 完整流程

1. **信息架构** - 页面结构、导航、信息层级
2. **结构设计** - 布局、栅格、内容组织
3. **交互流程** - 用户操作路径、状态转换
4. **UI稿图** - 低保真原型（可用 ASCII Art 或 Mermaid）
5. **设计规范** - 颜色、字体、间距等

### 判断标准

- 界面结构符合用户心智模型
- 交互路径清晰、无歧义
- UI稿图清晰传达设计意图

### 失败处理

- 信息过载 → 简化层级、分散步骤
- 交互复杂 → 重新设计路径
- UI稿图表达不清 → 补充注释或多个版本

### CONFIRM 节点

- 信息架构后：**"页面结构：{列表}，确认吗？"**
- 交互流程后：**"用户路径：{流程}，确认吗？"**
- UI稿图完成后：**"界面设计完成，确认后进入任务规划"**
