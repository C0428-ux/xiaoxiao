# 前置条件详解 | tdd-development

## 进入条件

- `task-planning` 状态为 `completed`
- 任务列表已确认

## 退出条件

- 所有 P0/P1 任务完成，用户确认

## 中断处理

- 中断时保存当前任务和进度
- `resume` 时询问从哪个任务继续
- TDD 内部循环不触发状态切换

## 依赖说明

- 前置：`task-planning`
- 输出：代码（按项目结构）

## 特殊规则

### 内部循环
- RED → GREEN → REFACTOR 循环不触发状态切换
- 每次循环更新 `skill.loopCount`
- 单任务最大循环次数：10
- 超时强制移交用户

### 测试覆盖率
- P0 功能必须有测试
- P1 功能鼓励有测试
- P2 功能可选
