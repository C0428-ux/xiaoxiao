# 流程详解 | ship

## Step 1: 代码审查

**检查项**:
- 测试覆盖率
- 代码规范
- 安全漏洞
- 性能问题

### 代码审查清单（来自 gstack）

#### SQL 安全
- [ ] 检查所有 SQL 查询是否使用参数化查询
- [ ] 检查是否有 SQL 拼接（`${}`, `+` 拼接）
- [ ] 检查 ORM 使用是否正确（无 N+1 查询泄漏）

#### LLM Trust Boundary（如果涉及 AI 功能）
- [ ] 检查 AI 输出是否经过验证
- [ ] 检查是否有可能的 prompt injection
- [ ] 检查敏感数据是否未发送到 AI

#### 条件副作用
- [ ] 检查条件分支是否有隐藏副作用
- [ ] 检查 null/undefined 是否被正确处理
- [ ] 检查错误处理是否完整

#### 结构性检查
- [ ] 检查是否有未处理的 Promise rejection
- [ ] 检查是否有内存泄漏（未清理的监听器）
- [ ] 检查是否有竞态条件

#### 测试覆盖
- [ ] 关键路径是否有测试
- [ ] 边界条件是否有测试
- [ ] 错误情况是否有测试

**问题分级**:
- 🚨 严重：必须修复
- ⚠️ 警告：建议修复
- 💡 建议：可选改进

---

## Step 2: QA 测试

在部署前进行系统性 QA 测试。

### QA 三层模式

| 模式 | 覆盖范围 | 适用场景 |
|------|----------|----------|
| **Quick** | Critical + High | 紧急修复、简单变更 |
| **Standard** | + Medium | 常规发布 |
| **Exhaustive** | + Cosmetic | 重要功能、新增功能 |

### QA 检查项

#### 功能测试
- [ ] 核心功能是否正常工作
- [ ] 用户流程是否顺畅
- [ ] 表单提交是否正常
- [ ] 页面导航是否正常

#### 边界测试
- [ ] 空输入处理
- [ ] 超长输入处理
- [ ] 特殊字符处理
- [ ] 并发操作

#### 错误处理
- [ ] 404/500 错误页面
- [ ] 网络错误提示
- [ ] 表单验证错误提示

#### 浏览器测试（如适用）
- [ ] Chrome 测试
- [ ] 移动端响应式

### QA 输出格式

```markdown
## QA 报告

### 测试模式：{Quick/Standard/Exhaustive}

### Health Score
- Before: X/10
- After: Y/10

### 问题列表
| 严重度 | 问题描述 | 状态 |
|--------|----------|------|
| 🚨 High | ... | Fixed/Pending |
| ⚠️ Medium | ... | Fixed/Pending |

### Ship Readiness
☐ Ready to deploy / ☐ Not ready
```

## Step 3: 部署准备

**检查清单**:
- [ ] 环境配置完成
- [ ] 依赖安装成功
- [ ] 数据库迁移就绪
- [ ] 备份完成

---

## Step 4: 执行部署

**做法**:
- 按部署文档执行
- 记录每一步结果
- 出现问题立即停止

---

## Step 5: 验证发布

**功能检查**:
- [ ] 核心功能正常工作
- [ ] 表单/交互正常
- [ ] 页面导航正常

**性能检查**（来自 gstack benchmark）:
- [ ] 页面加载时间合理（< 3s）
- [ ] Core Web Vitals 达标（LCP < 2.5s, FID < 100ms, CLS < 0.1）
- [ ] 无性能回归（与基准对比）

**错误检查**:
- [ ] 控制台无 Error 级别错误
- [ ] 无未处理的 Promise rejection
- [ ] API 响应正常

---

## Step 6: 监控上线

**持续监控指标**:
- 错误率
- 响应时间
- 资源使用（CPU/内存）

**性能趋势**:
- 记录基准值
- 对比历史数据
- 检测性能回归

---

## CONFIRM 话术

### 审查完成
> "代码审查完成。🚨 {N1} 个严重问题（已修复/待修复），⚠️ {N2} 个警告。"

### QA 完成
> "QA 完成。问题：🚨 {N1} 个 High，⚠️ {N2} 个 Medium。Health Score: {X}/10 → {Y}/10。确认进入部署吗？"

### 部署前
> "即将部署到 **{环境}**。回滚方案：{方案}。确认吗？"

### 上线后
> "部署成功。监控中... 指标正常吗？"
