# Ship | 发布上线

## Layer 1 - 入口

**触发词**: `/ship`, `发布`, `上线`, `deploy`

**前置条件**: tdd-development 已完成

**核心动作**:
1. 代码审查
2. 部署准备
3. 执行部署
4. 验证发布
5. 监控上线

---

## Layer 2 - 上下文

### 目标

安全、顺畅地将产品发布上线。

### 完整流程

1. **代码审查** - 检查代码质量、规范、安全
2. **部署准备** - 环境配置、依赖安装、数据库迁移
3. **执行部署** - 按部署流程执行
4. **验证发布** - 功能验证、性能验证
5. **监控上线** - 监控错误率、性能指标

### 判断标准

- 所有测试通过
- 部署无错误
- 功能验证通过
- 性能指标达标

### 失败处理

- 部署失败 → 回滚并分析原因
- 功能异常 → 紧急修复或回滚
- 性能不达标 → 优化或延迟上线

### CONFIRM 节点

- 代码审查后：**"代码审查通过，发现 {N} 个问题（已修复/待修复），确认部署吗？"**
- 部署前：**"即将部署到 {环境}，确认吗？"**
- 上线后：**"发布成功，监控中... 有异常吗？"**
