# 发布检查清单 | Release Checklist

> 发布前必须检查的所有项目

---

## 1. 代码质量

- [ ] 所有测试通过（本地 + CI）
- [ ] 代码 review 已通过
- [ ] 没有未解决的 lint 错误
- [ ] 安全扫描通过

## 2. 功能完整性

- [ ] 实现了所有计划的 feature
- [ ] 没有阻断性 bug
- [ ] 文档已更新
- [ ] 隐私政策合规（如需要）

## 3. 性能

- [ ] 响应时间符合 SLA
- [ ] 压力测试通过
- [ ] 没有内存泄漏
- [ ] 资源使用正常

## 4. 配置

- [ ] 环境变量正确配置
- [ ] Secrets 已配置（不在代码中）
- [ ] 域名已配置
- [ ] CDN 配置（如需要）

## 5. 数据

- [ ] 数据库迁移脚本已准备
- [ ] 回滚脚本已准备
- [ ] 种子数据已准备（如需要）
- [ ] 数据验证通过

## 6. 部署

- [ ] 部署脚本测试通过
- [ ] 回滚方案已验证
- [ ] 部署时间已确认
- [ ] 值班人员已安排

## 7. 监控

- [ ] 健康检查端点正常
- [ ] 告警规则已配置
- [ ] Dashboard 可查看
- [ ] 日志正常采集

## 8. 通知

- [ ] 相关团队已通知
- [ ] 用户通知（如需要）
- [ ] 支持团队已就绪
- [ ] 外部依赖方已通知

## 9. 准备回滚

- [ ] 上一版本仍可用
- [ ] 回滚脚本可执行
- [ ] 回滚时间已评估
